# Git Flow Rollout Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Configure `newsdom-api` to use a manual classic Git Flow model on GitHub without relying on git config changes.

**Architecture:** Introduce `develop` as the integration branch and GitHub default branch while keeping `main` as the stable release branch. Store branch workflow guidance in maintainer-facing docs and lightweight PR automation, not in the user-facing README.

**Tech Stack:** Git, GitHub, Markdown documentation.

---

**Files:**

- Modify: `README.md`

### Task 3: Create and publish `develop`

**Files:**
- No file changes.

**Step 1: Run command to create branch**

Run: `git checkout -b develop`

**Step 2: Push branch**

Run: `git push -u origin develop`

**Step 3: Set GitHub default branch**

Run: `gh repo edit Seongho-Bae/newsdom-api --default-branch develop`

**Step 4: Verify branch state**

Run:

```bash
git branch -a
gh repo view Seongho-Bae/newsdom-api --json defaultBranchRef
```

Expected: `develop` exists and is the default branch.
