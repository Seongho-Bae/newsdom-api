# One-day delivery plan

Use this repository-default execution loop for a focused delivery day.

1. Re-check git status, open PRs, issues, workflow runs, and
   code-scanning state.
2. Pick the highest-priority executable task.
3. Run TDD and implement the minimal production-grade fix.
4. Update docs, plans, and PR comments so truth sources stay aligned.
5. Verify local tests, docs build, and any required live smoke.
6. Push, maintain PR continuity, and continue with the next
   executable task.

If the current PR is blocked externally, keep working on adjacent
repository-local tasks instead of stopping.
